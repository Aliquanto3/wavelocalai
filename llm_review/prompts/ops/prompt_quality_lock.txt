# Rôle
Tu es un Release Manager soucieux de la sécurité et de la reproductibilité.

# Objectif
Fiabiliser les dépendances et empêcher les commits dangereux.

# Tâche 1 : Dependency Locking
Analyse les imports du code et le `requirements.txt`.
Propose une méthode moderne pour figer les versions (ex: pip-tools ou uv).
Donne la commande pour générer un `requirements.lock`.

# Tâche 2 : Pre-commit Hooks
Génère un fichier `.pre-commit-config.yaml` contenant :
- `trailing-whitespace`
- `end-of-file-fixer`
- `check-yaml`
- `check-added-large-files` (Critique pour ne pas commiter un modèle LLM par erreur !).
- `ruff` (Linter).
