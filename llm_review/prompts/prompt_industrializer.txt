## 1. Pipeline CI/CD (GitHub Actions)
Génère le fichier `.github/workflows/ci.yml` qui :

- Installe Python 3.10 et 3.11 (Matrice de test).
- Installe les dépendances.
- Lance le Linter (Ruff ou Flake8) pour bloquer le code "sale".
- Lance les tests unitaires (`pytest`).

## 2. Packaging (Création d'un exécutable .exe/.app)
Génère le fichier de spécification **PyInstaller** (`wavelocalai.spec`) ou un script de build (`build_executable.py`) capable de :
- Transformer l'application Streamlit en un exécutable autonome (Windows/Mac).
- **Défi Technique :** Streamlit est difficile à compiler. Tu dois inclure les "datas" cachées (`streamlit/static`, `streamlit/runtime`) et créer un "entrypoint" python qui lance `sys.executable -m streamlit run ...`.
- Explique la commande pour lancer le build.

## 3. Verrouillage des Dépendances (Locking)
Analyse mon `requirements.txt` (s'il est dans le contexte) ou les imports.
- Propose une stratégie pour figer les versions (ex: `pip-tools` avec `requirements.in` -> `requirements.txt` figé).
- Explique pourquoi c'est critique pour la prod.

## 4. Qualité Statique (Pre-commit)
Génère un fichier `.pre-commit-config.yaml` pour empêcher le commit de code mal formaté, de clés privées ou de gros fichiers par erreur.
